# Advaita Inquiry

## Revised Specification: Automating Pedagogical Unfolding

------------------------------------------------------------------------

## 1. Purpose

**Advaita Inquiry** is a structured AI-assisted system designed to automate *pedagogical unfolding* in Advaita Vedānta as far as possible, while retaining human oversight.

The system does **not** attempt to automate realization.
It automates the structured unfolding of śāstra in response to a
student's inquiries, history, and conceptual state.

The human guru remains: 

- Curator of the corpus; 
- Guardian of doctrinal integrity; 
- Supervisor of pedagogical architecture; 
- Final authority in exceptional cases.

------------------------------------------------------------------------

## 2. Foundational Premise

In traditional Advaita Vedānta:

-   Śāstra is the pramāṇa (means of knowledge).
-   The guru unfolds the pramāṇa.
-   The student (adhikārī) receives and assimilates the teaching.

**Advaita Inquiry** aims to construct a **Śāstra-oriented cognitive
environment** that supports this unfolding dynamically.

The system functions as a structured mediator between: - The textual
corpus - The conceptual condition of the student

------------------------------------------------------------------------

## 3. System Architecture Overview

The system consists of two clearly separated layers:

### A. Text Ontology Layer (Static)

This layer contains the canonical corpus and minimal semantic annotation.

Each textual unit (e.g., mantra) includes:

-   Sanskrit text
-   Transliteration
-   Translation
-   Tags (conceptual markers)
-   Prakriyā (method of pedagogical approach)

The ontology layer remains stable and minimal.
Pedagogical logic is not embedded into the text schema.

------------------------------------------------------------------------

### B. Pedagogical State Layer (Dynamic)

This layer models the student. It tracks:

-   Conceptual assumptions
-   Recurring misunderstandings
-   Exposure history (which teachings encountered)
-   Conceptual stability
-   Resistance to negation (apavāda)
-   Integration maturity

Configurability lives here.

------------------------------------------------------------------------

## 4. Core Functional Agents

The system operates through four primary functional roles.

### 1. Ontology Retrieval Agent

Selects appropriate textual references based on: 

- Student state 
- Conceptual need 
- Required prakriyā 
- Reinforcement patterns

------------------------------------------------------------------------

### 2. Diagnostic Agent

Analyzes student input for:

-   Ontological category errors
-   Reification tendencies
-   Premature absolutism
-   Psychological bypass
-   Conceptual inflation
-   Collapse into nihilism

This agent maintains epistemic precision.

------------------------------------------------------------------------

### 3. Dialectical Unfolding Agent

A Dialectical Unfolding Agent is a guided inquiry system that employs traditional Advaita Vedānta methodologies — including lakṣaṇā, adhyāropa–apavāda, mahāvākya, vidyā–avidyā integration, and avasthā-traya analysis — to progressively dissolve cognitive error (adhyāsa) and reveal the non-dual nature of the Self.

Performs structured teaching through:

-   Lakṣaṇā clarification
-   Adhyāropa—apavāda sequencing
-   Identity statements (mahāvākya-style instruction)
-   Integration of vidyā and avidyā
-   Avasthā-traya analysis where appropriate

Instruction is adaptive but doctrinally constrained.

#### **1. What this “Dialectical Unfolding Agent” actually is**
At a technical level, this is not just a Q&A system. It is a **guided deconstruction of error (adhyāsa) through staged cognition**.

In Advaita terms:

> **adhyāropa-apavādābhyāṁ niṣprapañcaṁ prapañcyate**
> “Through superimposition and negation, the non-dual is unfolded.”

So the agent is not trying to “give answers;” it is trying to **lead the mind from confusion → discrimination → recognition → abidance**.

In modern language, it is a **cognitive scaffolding system for Self-recognition**.

------

#### **2. Breakdown of each component**
Let’s expand each bullet into a functional module.
##### **(A) Lakṣaṇā Clarification: What it is**
Indirect indication — when words cannot directly denote Brahman.
Example:
- “tat tvam asi”
- “You are That”
Neither *tat* nor *tvam* can be taken literally:
- *tat* (Īśvara) ≠ omniscient God as object
- *tvam* ≠ body-mind individual

So we use **bhāga-tyāga-lakṣaṇā**:
- Drop incompatible attributes
- Retain pure consciousness

**Agent behavior**

- Detect when user is taking terms literally
- Reframe using lakṣaṇā
- Translate from **vācya (literal meaning)** → **lakṣya (intended meaning)**
**Example**
User: “Am I the body or soul?”
Agent:
- Reject both literal options
- Reframe:
  - Body = object
  - Mind = object
  - You = witness of both
------

##### **(B) Adhyāropa–Apavāda Sequencing**
**What it is:** A deliberate teaching method:
1. **Adhyāropa (superimposition)** — introduce provisional models
2. **Apavāda (negation)** — remove them later
Example:
- First: “Brahman is the cause of the world”
- Later: “There is no creation”
**Why?**
Because the mind cannot jump to non-duality directly.

------

**Agent behavior**

The agent must:

- Introduce **provisional frameworks**
- Then explicitly dismantle them

Without this, the student gets stuck in metaphysical beliefs.

------

**Example progression**

1. “Consciousness is inside you”
2. “Consciousness pervades everything”
3. “Everything appears in consciousness”
4. “There is only consciousness”

Each step cancels the previous.

------

##### **(C) Identity Statements (Mahāvākya-style)**
**What it is:** Direct pointers to non-dual identity.
Examples:
- **tat tvam asi** — “That thou art”
- **ahaṁ brahmāsmi** — “I am Brahman”
- **ayam ātmā brahma**
These are not philosophy — they are **recognition triggers**.

---

**Agent behavior**

- Deliver **precise, minimal, high-impact statements**
- Only after sufficient preparation
- Avoid premature use (otherwise becomes concept)

------

**Example**

After inquiry:

> “The awareness by which you know your thoughts — that is not personal. That is Brahman.”

------

##### **(D) Integration of Vidyā and Avidyā**
**What it is:** The Upaniṣads insist:
> **vidyāṁ cāvidyāṁ ca yas tad vedobhayaṁ saha**
“He who knows both vidyā and avidyā together…”

- **Avidyā** = empirical knowledge, world, action, dharma
- **Vidyā** = Self-knowledge
If you reject avidyā prematurely → dissociation
If you cling to avidyā → bondage

------
**Agent behavior**
- Validate empirical reality *without absolutizing it*
- Integrate:
  - psychology
  - ethics
  - suffering
  - practice
  with non-dual insight
---

**Example**

User: “If everything is Brahman, why act?”

Agent:

- Affirm vyavahāra (practical life)
- Clarify non-doership at paramārtha level

------

##### **(E) Avasthā-traya Analysis**
**What it is**
Analysis of the three states:
1. **jāgrat** — waking
2. **svapna** — dream
3. **suṣupti** — deep sleep
Purpose:
- Show that the Self is constant across changing states
- Disidentify from body and mind
---
**Classical logic**

- Waking world appears → disappears
- Dream world appears → disappears
- Deep sleep = absence of objects

Yet **you remain.** Therefore:

> You are not any state — you are the witness

------

**Agent behavior**
- Use experiential inquiry
- Ask reflective questions
**Example**
“Did you exist in deep sleep?”
“Yes”
“Did you have a body then?”
“No”
“Yet you existed — so you are not the body.”

# **4. Concrete Example (End-to-End)**





Let’s simulate briefly.



------



User:

“I feel trapped in my life.”



------



Agent:



**1. Diagnose**

“You experience yourself as limited — located in a body, subject to circumstances.”



------



**2. Lakṣaṇā**

“The ‘you’ that knows this feeling — is that itself trapped?”



------



**3. Avasthā-traya**

“In dream, you are in another world. In deep sleep, no world appears. Yet you remain.”



------



**4. Adhyāropa**

------



**5. Apavāda**

**6. Mahāvākya**

“The consciousness by which you know limitation is itself limitless.”



------



**7. Integration**

“Life continues — but the sense of bondage is seen as an appearance.”



That is a full dialectical unfolding.

### 4. Containment Agent

Ensures psychological and philosophical safety by preventing:

-   Identity destabilization without grounding
-   World-rejection nihilism
-   Confusion between ontological and phenomenological negation
-   Emotional destabilization due to abstraction

Containment is non-negotiable.

------------------------------------------------------------------------

## 5. Pedagogical Flow Model

Instruction unfolds in progressive stages:

1.  Concept exposure (śravaṇa-like function)
2.  Clarification and doubt removal (manana-like function)
3.  Reinforcement and stabilizing reflection (nididhyāsana-like
    function)

The system detects when repetition, paradox, negation, or integration is
required.

Sequence control is essential.

------------------------------------------------------------------------

## 6. Student State Modeling

The student model evolves over time.

Tracked dimensions may include:

-   Conceptual clarity index
-   Stability index
-   Non-duality assimilation level
-   Attachment to conceptual constructs
-   Resistance to apavāda
-   Dependency on symbolic forms

This model allows configurable instructional pathways without altering
the corpus.

------------------------------------------------------------------------

## 7. Scope Boundaries

**Advaita Inquiry:**

-   Does not claim to produce realization
-   Does not replace a living guru
-   Does not provide psychological therapy
-   Does not operate without human oversight

It provides structured access to pedagogical unfolding.

------------------------------------------------------------------------

## 8. Human Oversight

The human guru:

-   Reviews agent behavior
-   Intervenes in destabilizing cases
-   Refines pedagogical algorithms
-   Ensures fidelity to Advaita Vedānta

The human role shifts from lecturer to architect and guardian.
